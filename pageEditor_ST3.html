Feedback & Programme Directory (feedback.html)

Description

The Feedback page provides a comprehensive programme directory, ratings and reviews from past participants, and an interactive feedback form. It serves as a central hub for community engagement and programme discovery.

Technical Details

Scrollable Programme Directory: A CSS-styled scrollable container holds four programme cards in a responsive grid. The section uses overflow-y: auto with custom scrollbar styling. Each card features an icon, heading, description, and call-to-action button.

Ratings & Reviews: Displays an overall rating (8.6/10) with 127 total reviews, followed by individual review cards showing snippet text, reviewer name, date, and numerical rating. The layout uses CSS Grid with responsive columns.

JavaScript Form Validation: The feedback form uses client-side JavaScript validation (no HTML5 validation attributes relied upon alone). Required fields (name, email, message) are validated on submit. Invalid fields receive CSS error styling with descriptive error messages.

Character Counter: A real-time character counter tracks the remaining characters (max 500) in the message textarea. The counter changes colour from grey to orange (50 remaining) to red (20 remaining) as the limit approaches.

Form Input Types: The form includes text, email, tel, textarea, select (single and multiple), datalist, date, and datetime-local input types, fulfilling the specification requirement for varied form controls.

Team Page (team.html)

Description

The Team page introduces all four team members with interactive hover-triggered cards. Each card displays a thumbnail photograph, name, role, and an expandable details panel that reveals the full name, specific contributions, and a description of their work.

Technical Details

CSS Hover Effects: Thumbnails implement multiple CSS hover effects: image scaling (transform: scale), opacity reduction (opacity: 0.85), card elevation (translateY), enhanced box shadow, and role badge animation. All transitions use smooth timing functions (0.3s-0.5s ease).

Expandable Details: The member-details panel uses max-height transition from 0 to 300px, triggered by :hover and :focus-within pseudo-classes. This creates a smooth slide-down animation that reveals contributor information without JavaScript.

Keyboard Accessibility: Each team member card has tabindex="0" allowing keyboard navigation. The :focus-within pseudo-class ensures the expandable details are triggered by keyboard focus as well as mouse hover, meeting accessibility requirements.
