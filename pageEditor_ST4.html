User Profile - Progressive Builder (profile.html)

Description

The User Profile page allows visitors to build their climate action profile progressively using JavaScript prompts (no HTML forms). The system guides users through three steps with three prompts each, tracks completion percentage, and displays the collected information dynamically in the page DOM.

Technical Details

JavaScript Prompts: The system uses window.prompt() to collect user input across 9 questions divided into 3 steps: Basics (name, username, local area), Interests (SDG topic, preferred content, weekly time), and Actions (current action, future goal, climate pledge). Users can leave prompts blank to skip them.

Progressive Disclosure: Profile sections are hidden by default and only appear once a step has data. New data is animated into view using CSS keyframe animations (slideIn). The profile output area updates dynamically via DOM manipulation without page reload.

Completion Tracking: A progress bar with percentage indicator tracks how many of the 9 prompts have been answered. The bar uses CSS width transitions for smooth animation. A prompt count indicator shows x/9 prompts completed.

Step Navigation: Step tabs show current, completed, and pending states with colour coding. Revisit buttons allow users to return to any step and update or fill in previously skipped prompts. The "Start / Continue Profile" button intelligently finds the next incomplete step.

Confirmation: When all 9 prompts are answered, a confirmation section appears with a congratulatory message. The reset button clears all data after a confirm() dialog.

Interactive SVG Sitemap (sitemap.html)

Description

The Sitemap page presents a visual, hierarchical representation of the entire ClimAct website structure using an inline SVG graphic. Every page in the website is represented as a clickable node that navigates directly to that page.

Technical Details

SVG Structure: The sitemap is built as an inline SVG with viewBox for responsive scaling. The hierarchy includes 5 levels: Entry Point (Splash Screen), Main Page (Home), Navigation Pages (7 pages), Content Pages (4 individual pages), and Supporting Pages (Page Editors and Validation Reports).

Clickable Nodes: Each node is wrapped in an SVG anchor (<a>) element with href pointing to the respective HTML file. This provides native clickable functionality without JavaScript.

Hover Feedback: CSS rules target SVG anchor children (rect and ellipse) to provide visual feedback on hover: reduced brightness and increased stroke width. This indicates interactivity to users.

Accessibility: Each SVG anchor contains a <title> element providing descriptive text for screen readers. The overall SVG has role="img" and aria-label attributes. Focus states are styled with a visible orange outline.

Colour Coding: Nodes are colour-coded by type: dark green (main page), medium green (navigation pages), light green (content pages), orange (page editors), grey (validation pages), and amber (entry point). A legend at the bottom explains the colour scheme.
